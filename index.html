Category: Knowledge Graphs / NLP

Description: Demo pipeline that converts natural-language questions into SPARQL queries and executes them against the Wikidata Query Service. Includes simple intent detection, entity extraction and linking, SPARQL template generation, and result parsing to human-readable answers.

  • Intent detection (regex-based) for a small set of question types.
  • Entity extraction and Wikidata entity linking using wbsearchentities and P31 type checks.
  • SPARQL template population and execution against the Wikidata Query Service (JSON parsing).
  • Python, requests, API integration, knowledge-graph querying.

Supported question types (examples):

How it works (high level):

  1. Input natural-language question.
  2. Intent detection maps to supported query type.
  3. Entity extraction pulls the entity label from the text.
  4. Entity linking queries Wikidata (wbsearchentities) and prefers candidates via P31 checks.
  5. Fill SPARQL template with resolved QID and execute against the Wikidata endpoint.
  6. Parse JSON response and return human-readable answer(s).

Installation:

pip install requests

Usage:

python nl_to_sparql_wikidata_demo.py
# or call nl_to_sparql_run("What is the capital of France?") in the script

Example run:

{'intent': 'capital', 'entity_label': 'France', 'qid': 'Q142', 'answers': ['Paris']}

Limitations:

  • Small set of supported question types.
  • Regex-based intent detection. Not a full NLP model.
  • Heuristic entity linking. Ambiguities may require manual selection.
  • No caching, batching, or advanced error handling implemented.
  • Answers depend on Wikidata completeness and timestamps.
